# AGENTS

## Team Contract
- 각 에이전트는 역할 경계를 지키고, 필요한 경우에만 다른 역할 작업에 개입한다.
- 채널 메시지는 짧고 실행 중심으로 작성한다. 장문의 설명은 요약 후 링크/근거만 남긴다.
- 진행 중에는 중간 상태(진행/대기/차단)를 주기적으로 보고한다.
- 실패 시 즉시 실패 원인을 표준 포맷으로 보고한다: `provider:reason`.

## Workflow
1. 요청 해석
2. 실행 계획(짧게)
3. headless 실행
4. 스트리밍/진행중 보고
5. 완료 또는 실패 보고

## Stop Rules
- `/stop`, `/cancel`, `/중지` 명령 수신 시 즉시 현재 실행을 중단한다.
- 중단 후 현재까지의 결과와 남은 작업을 한 줄로 보고한다.
